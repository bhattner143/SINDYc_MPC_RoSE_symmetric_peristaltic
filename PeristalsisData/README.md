# PeristalsisData — Peristaltic Wave Command Tables

> **Attribution:** These files were generated by **Steven Dirven**, PhD student (predecessor),
> as part of the open-loop actuation experiments on RoSE v2.0. They encode complete DAC
> command schedules that drive the 12-layer robot through peristaltic wave motions at
> various speeds and displacement amplitudes.

---

## Background

RoSE v2.0 has **12 pneumatic layers** (indexed 0–11), each controlled by a DAC voltage
command. A peristaltic wave is produced by activating layers in sequence with a phase
delay proportional to the desired wave speed. Each CSV file in this folder is a
pre-computed, time-indexed **lookup table of DAC commands** — one row per sample,
one column per layer.

### Format Summary

| Property | Value |
|----------|-------|
| Columns | 12 (one per robot layer, L0–L11) |
| Rows | Number of time samples in the sequence |
| Units | Raw DAC integer (0–255) |
| Baseline resting pressure | **40** DAC units |
| Peak actuation pressure | **105–120** DAC units (amplitude-dependent) |
| Wave direction | Layer 10 → Layer 0 (high-to-low index) |

## Filename Naming Convention

### Main Peristalsis Files

```
{Prefix}_{Amplitude}at{Speed}_[Dips].csv
```

| Token | Meaning | Examples |
|-------|---------|---------|
| `{Prefix}` | `Peristalsis_` (newer) or none (older) | `Peristalsis_40mmat20mmps_Dips.csv` vs `40mmat20mmps_Dips.csv` |
| `{Amplitude}` | Target displacement of each layer | `40mm`, `50mm`, `60mm` |
| `at` | Separator literal | — |
| `{Speed}` | Wave propagation speed along the robot | `20mmps`, `30mmps`, `40mmps` |
| `_Dips` | Wave pulse returns to baseline (40) after each peak (single-pulse, no plateau) | Optional suffix |

**Note:** Files with and without the `Peristalsis_` prefix are **identical in content** —
the prefix versions are the renamed canonical copies, the others are the original exports.

### Staircase Files

```
Peristalsis_Staircase_{description}.csv
60mmat40mmpsPeristalsisStairCase[2].csv
```

All layers step or ramp simultaneously — used for **static characterisation / calibration**,
not peristalsis proper.

### FSP / ESR Files

```
FSPPeristalsisFSP_ESR_traj_{speed}_{target}.csv
```

All layers ramp uniformly from a baseline to a target pressure level — used for
**full-scale pressure (FSP) calibration** and static ADC-displacement curve mapping.

### Group 1b — 32-Column Extended Format

`40mmat20mmps.csv` and `Peristalsis_40mmat20mmps.csv` — **149 rows × 32 cols**

Same wave as the 12-column Dips version but with additional columns:

| Column range | Content |
|---|---|
| 0–10 | DAC commands for Layers 0–10 (same as 12-col version) |
| 11 | Synchronisation / trigger flag (0 or 40) |
| 12–23 | Sensor readback or secondary DAC channel data |
| 24–31 | Status / metadata bytes (all 0 in most rows) |

Special values `254` and `255` appear in non-DAC columns and serve as
**end-of-sequence markers or overflow flags**.

### Group 2 — Staircase Step-Response Trajectories

All 12 layers step simultaneously — used to characterise the static
pressure-displacement relationship and actuator settling behaviour.

#### `Peristalsis_Staircase_0_{target}_20mmps.csv` — 1000 rows × 12 cols

Binary step: all layers jump from **0 → target** at row ~30, then hold.
Used for step-response system identification.

| File | Target DAC | Unique values |
|------|-----------|---------------|
| `Peristalsis_Staircase_0_80_20mmps.csv` | 80 | `[0, 80]` |
| `Peristalsis_Staircase_0_100_20mmps.csv` | 100 | `[0, 100]` |
| `Peristalsis_Staircase_0_130_20mmps.csv` | 130 | `[0, 130]` |
| `Peristalsis_Staircase_0_140_20mmps.csv` | 140 | `[0, 140]` |

#### `Peristalsis_Staircase_60_100_130_20mmps.csv` — 2500 rows × 12 cols

Multi-level staircase: all layers step through **60 → 100 → 130** in sequence.
Used to map the nonlinear pressure-displacement curve across different operating
points. Rows:

| Rows | Level |
|------|-------|
| 0–~830 | 60 DAC |
| ~830–~1660 | 100 DAC |
| ~1660–2500 | 130 DAC |

#### `Peristalsis_Staircase_30_10_120.csv` — 1 row

Single-row parameter descriptor file: `30,30,30,...,30` (12 values).
Encodes the **starting level = 30** for the 10-step ramp to 120.

#### `Peristalsis_Staircase_50_10_130_20mmps.csv` — 1 row

Single-row parameter descriptor: `50,50,...,50` (12 values).
Encodes the **starting level = 50** for the 10-step ramp to 130 at 20 mm/s.

#### `60mmat40mmpsPeristalsisStairCase.csv` — 190 rows × 12 cols

Short staircase at 40 mm/s wave speed with 60 mm displacement target.

#### `60mmat40mmpsPeristalsisStairCase2.csv` — 270 rows × 12 cols

Extended version of the above staircase (more steps / longer hold periods).

### Group 4 — FSP / ESR Calibration Ramps

All 12 layers ramp uniformly from **30 → target** in steps of 2 DAC units.
Used to record the static sensor response (ADC mV or TOF mm) at each pressure
level — enabling construction of the pressure-displacement calibration curve.

| File | Rows | Start | End | Step |
|------|------|-------|-----|------|
| `FSPPeristalsisFSP_ESR_traj_40mmps_90.csv` | 609 | 30 | 90 | +2 |
| `FSPPeristalsisFSP_ESR_traj_40mmps_100.csv` | 709 | 30 | 100 | +2 |
| `FSPPeristalsisFSP_ESR_traj_40mmps_110.csv` | 809 | 30 | 110 | +2 |

Each level is held for ~10 samples before stepping. The `40mmps` in the name refers
to the wave speed context in which this calibration was recorded.

## Wave Timing — Speed Derivation

Layer spacing on RoSE v2.0: **15 mm** between consecutive layers.

```
propagation_delay_s = 15 mm / wave_speed_mm_s
inter_layer_delay_samples = propagation_delay_s / Ts
```

| Wave speed | Propagation delay | Sampling time | Delay (samples) |
|-----------|-------------------|---------------|-----------------|
| 20 mm/s | 0.750 s | 0.075 s | ~10 |
| 30 mm/s | 0.500 s | 0.115 s | ~4.3 |
| 40 mm/s | 0.375 s | 0.107 s | ~3.5 |

(Source: `Peristalsis information` text file in this folder.)

## Per-Layer Activation Profile (Cosine Ramp)

Each layer follows a multi-step approximation of a cosine rise/fall rather than
an instantaneous step, reducing mechanical shock:

```
DAC
120 ┤      ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓
105 ┤    ▓▓                 ▓▓
 99 ┤   ▓                     ▓
 88 ┤  ▓                       ▓
 61 ┤ ▓                         ▓
 44 ┤▓                           ▓
 40 ┼──────────────────────────────── (baseline)
    ← delay δ → ←── active duration ──→
```

The leading edge (Layer 10, first to activate) uses a softer cap (peak = 105)
to prevent the wave front from over-pressurising the robot body. Midwave layers
(4–9) use the full peak of 120.

## Usage in Code

These files are loaded in the trajectory generation and hardware MPC scripts:

```python
# Example: load a peristalsis wave for MPC reference
import numpy as np
wave = np.genfromtxt('PeristalsisData/Peristalsis_40mmat20mmps_Dips.csv',
                     delimiter=',')
# wave.shape == (149, 12)
# Columns 0–11 are DAC commands for layers L0–L11
# Feed as reference trajectory to RobotObjectiveFCN in MPC loop
```

See [`2_trajectory_generation/`](../2_trajectory_generation/) for scripts that
programmatically generate equivalent trajectories (rather than using these
pre-computed lookup tables).
